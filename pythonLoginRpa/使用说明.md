# Codex 批量登录工具 使用说明

## 目录结构

```
pythonLoginRpa/
  codex_batch_login.py     主脚本
  requirements.txt         Python 依赖
  config.example.json      配置文件模板
  config.json              实际配置文件（需自行创建，已被 .gitignore 排除）
  .gitignore               排除敏感文件，不提交到 git
```

---

## 工作原理

```
Python 脚本                     CLIProxyAPI 服务器              OpenAI
─────────────────────────────────────────────────────────────────────
1. GET /codex-auth-url    →     生成授权 URL + state
                          ←     返回 {url, state}

2. Playwright 打开 url，自动填写邮箱/密码
   OpenAI 登录成功后跳转 → http://localhost:1455/auth/callback?code=...&state=...
   Playwright 拦截该跳转，获取完整 URL（不需要 1455 端口真正监听）

3. POST /oauth-callback   →     写入回调文件
                                后台自动完成 code→token 交换
                                凭证 JSON 保存到 auth-dir
```

## 环境准备

### 1. Python 版本

需要 Python 3.8 及以上。

```bash
python --version
```

### 2. 安装依赖

```bash
cd pythonLoginRpa
pip install -r requirements.txt
playwright install chromium
```

如果账号启用了双重认证（TOTP / 2FA），额外安装：

```bash
pip install pyotp
```

## 配置文件

将 `config.example.json` 复制为 `config.json`，然后按需修改：

```bash
copy config.example.json config.json
```

`config.json` 各字段说明：

```json
{
  "mgmt_url": "http://127.0.0.1:8317",
  "mgmt_key": "your-secret-key-here",
  "headless": true,
  "delay_between_accounts": 3,
  "accounts": [
    {
      "email": "account1@gmail.com",
      "password": "password1",
      "totp_secret": ""
    }
  ]
}
```

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `mgmt_url` | CLIProxyAPI 管理接口地址 | — |
| `mgmt_key` | 管理密码（对应 config.yaml 中的 `secret-key`） | — |
| `headless` | `true` 无头模式（后台运行）；`false` 显示浏览器窗口 | `true` |
| `delay_between_accounts` | 每个账号之间的等待秒数，避免频率限制 | `3` |
| `accounts` | 账号列表 | — |
| `accounts[].email` | OpenAI 账号邮箱 | — |
| `accounts[].password` | 账号密码 | — |
| `accounts[].totp_secret` | TOTP 密钥（Base32），没有双重认证填空字符串 | `""` |

### CLIProxyAPI 服务器的必要配置

`config.yaml` 中需要配置管理密码，本地运行时确保服务已启动：

```yaml
remote-management:
  secret-key: "your-secret-key-here"   # 与 mgmt_key 一致
  allow-remote: false                   # 本地运行保持 false 即可
  disable-control-panel: false
```

## 常见问题排查

### 问题 1：登录超时，生成了 debug_xxx.png 截图

打开截图查看浏览器停在哪一步：

- 停在邮箱/密码输入页 → 页面选择器可能已变更，将 `headless` 改为 `false` 观察
- 停在人机验证（CAPTCHA）→ 参考下方"验证码处理"章节
- 停在授权确认页 → 在截图中找到"Allow"或"授权"按钮的选择器，手动在脚本中补充点击逻辑

```bash
# 先用有头模式测试单个账号
# 修改 config.json: "headless": false, "accounts": [单个账号]
python codex_batch_login.py
```

### 问题 2：HTTP 403 错误

检查 CLIProxyAPI `config.yaml`：

```yaml
remote-management:
  secret-key: "与 mgmt_key 一致"
  allow-remote: true      # 如果从外网访问服务器，需要改为 true
```

### 问题 3：HTTP 500 / failed to start callback server

原因：端口 1455 已被占用（通常是上一次登录会话未释放）。

解决：

```bash
# Windows 查找占用 1455 端口的进程
netstat -ano | findstr :1455
taskkill /PID <PID> /F
```

或等待约 5 分钟让旧会话自动超时后重试。

### 问题 4：账号有双重认证（TOTP）

1. 安装 `pyotp`：`pip install pyotp`
2. 在 `config.json` 中填写 TOTP 密钥（Base32 格式，通常在扫描二维码时可以查看原始密钥）

```json
{
  "email": "xxx@gmail.com",
  "password": "xxx",
  "totp_secret": "JBSWY3DPEHPK3PXP"
}
```

### 问题 5：触发 Cloudflare 或人机验证

无头模式更容易被检测，可尝试：

```bash
pip install playwright-stealth
```

然后在 `codex_batch_login.py` 中 `browser_login_and_capture` 函数内的
`page = await context.new_page()` 后面添加：

```python
from playwright_stealth import stealth_async
await stealth_async(page)
```

## 注意事项

1. **不要提交 `config.json`**：该文件已加入 `.gitignore`，包含明文密码，请勿手动 `git add`
2. **账号间隔建议不低于 3 秒**：过快登录可能触发 OpenAI 的频率限制
3. **凭证自动保存**：登录成功后，JSON 凭证文件会由 CLIProxyAPI 服务器保存到 `auth-dir`（默认 `~/.cli-proxy-api/`），无需手动操作
4. **Token 自动续期**：凭证保存后由 CLIProxyAPI 负责自动刷新，无需重复登录
